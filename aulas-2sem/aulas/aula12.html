Testes E2E na Pirâmide de Testes

A pirâmide de testes indica a proporção ideal entre tipos de testes:

  • Testes Unitários: Base da pirâmide, rápidos e confiáveis.
  • Testes de Integração: Validam interações entre componentes.
  • Testes E2E (End-to-End): Validam o fluxo completo, simulando o usuário real.

Embora essenciais, testes E2E são mais lentos e frágeis, por isso devem ser usados com moderação.

Page Object Pattern / Model

É um padrão de projeto que encapsula a estrutura das páginas da aplicação, isolando a lógica de localização e interação dos elementos.

  • Facilita a manutenção dos testes.
  • Promove reutilização de código.
  • Melhora a leitura e organização dos testes.
// Exemplo Java com Selenium
public class LoginPage {
  private WebDriver driver;
  private By usernameField = By.id("username");
  private By passwordField = By.id("password");
  private By loginButton = By.id("loginBtn");

  public LoginPage(WebDriver driver) {
    this.driver = driver;
  }

  public void enterUsername(String username) {
    driver.findElement(usernameField).sendKeys(username);
  }

  public void enterPassword(String password) {
    driver.findElement(passwordField).sendKeys(password);
  }

  public void clickLogin() {
    driver.findElement(loginButton).click();
  }
}

Visual Regression Testing

Testes de regressão visual comparam screenshots da UI para detectar mudanças visuais não intencionais.

  • Importante para detectar bugs visuais após alterações.
  • Ferramentas populares: Applitools, Percy, BackstopJS.
  • Integração com Cypress, Selenium e Playwright.

Flaky Tests e Como Evitar

Flaky tests são testes que falham intermitentemente sem mudanças no código, causando insegurança e retrabalho.

  • Causas comuns:
    • Dependência de tempo / sincronização incorreta
    • Dados de teste inconsistentes
    • Problemas ambientais (rede, banco, etc.)
  • Como evitar:
    • Usar waits explícitos e evitar sleeps
    • Isolar dados e manter ambiente estável
    • Revisar e refatorar testes regularmente

Caso Real: Mercado Livre e Cypress

O Mercado Livre adotou Cypress para automatizar testes do fluxo de checkout com resultados expressivos.

  • Benefícios:
    • Testes rápidos e confiáveis
    • Integração contínua facilitada
    • Melhor cobertura do fluxo de compra
  • Abordagem:
    • Padrão Page Object para organizar testes
    • Testes simulam usuário final, incluindo pagamento
    • Uso de mocks para serviços externos quando necessário

Esse caso mostra como empresas grandes melhoram qualidade com testes E2E modernos.

Exercício Hands-on

Escrevendo teste E2E com Cypress para formulário web

Objetivo: Criar um teste Cypress que preencha um formulário, submeta e valide o resultado.

  1. Crie um arquivo de teste Cypress (ex: form.spec.js).
  2. Use o comando cy.visit() para abrir a página do formulário.
  3. Preencha os campos de texto (nome, email) usando cy.get().type().
  4. Selecione opções, se houver, com cy.get().select().
  5. Submeta o formulário com cy.get().click() no botão enviar.
  6. Valide a mensagem de sucesso com cy.contains() ou cy.get().should().
// Exemplo básico Cypress
describe('Formulário de Contato', () => {
  it('Deve preencher e submeter o formulário com sucesso', () => {
    cy.visit('/formulario');

    cy.get('#nome').type('João Silva');
    cy.get('#email').type('joao.silva@email.com');
    cy.get('#assunto').select('Feedback');
    cy.get('#mensagem').type('Ótimo site!');

    cy.get('#btnEnviar').click();

    cy.contains('Obrigado pelo seu contato!').should('be.visible');
  });
});

Dicas para o Exercício

  • Utilize seletores únicos para evitar flakiness.
  • Evite usar cy.wait(), prefira comandos implícitos do Cypress.
  • Organize comandos em funções para reutilização (exemplo do Page Object).
  • Execute testes no modo interativo para depurar facilmente.

Exemplo Java / Spring Boot - Teste E2E

// Teste E2E com Spring Boot e Selenium
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
public class CheckoutE2ETest {
  @Autowired
  private WebDriver driver;

  @Test
  public void testCheckoutFlow() {
    driver.get("http://localhost:8080/checkout");

    driver.findElement(By.id("address")).sendKeys("Rua Exemplo, 123");
    driver.findElement(By.id("paymentMethod")).click();
    driver.findElement(By.id("submitOrder")).click();

    WebElement confirmation = driver.findElement(By.id("confirmationMessage"));
    Assertions.assertTrue(confirmation.isDisplayed());
    Assertions.assertEquals("Pedido realizado com sucesso!", confirmation.getText());
  }
}

Exemplo .NET / C# - Teste E2E com Selenium

// Teste E2E com Selenium WebDriver em C#
[TestFixture]
public class LoginE2ETest {
  private IWebDriver driver;

  [SetUp]
  public void Setup() {
    driver = new ChromeDriver();
  }

  [Test]
  public void DeveLogarComSucesso() {
    driver.Navigate().GoToUrl("http://localhost:5000/login");

    driver.FindElement(By.Id("username")).SendKeys("usuario");
    driver.FindElement(By.Id("password")).SendKeys("senha123");
    driver.FindElement(By.Id("btnLogin")).Click();

    var welcome = driver.FindElement(By.Id("welcomeMessage"));
    Assert.IsTrue(welcome.Displayed);
    Assert.AreEqual("Bem-vindo, usuario", welcome.Text);
  }

  [TearDown]
  public void TearDown() {
    driver.Quit();
  }
}
