Modelos de Cloud Computing

  • IaaS (Infrastructure as a Service) - Infraestrutura como serviço: servidores, storage, rede na nuvem
  • PaaS (Platform as a Service) - Plataforma pronta para desenvolvimento e deploy
  • FaaS / Serverless - Funções executadas sob demanda, sem gerenciar servidores
  • SaaS (Software as a Service) - Aplicações completas entregues pela nuvem

CNCF Landscape

A Cloud Native Computing Foundation (CNCF) mantém um ecossistema vasto de projetos para computação nativa na nuvem.

  • Kubernetes - Orquestração de containers
  • Prometheus - Monitoramento
  • Envoy - Proxy e service mesh
  • Helm - Gerenciamento de pacotes Kubernetes
  • ...e muitos outros projetos para cloud native

Microsoft Azure - Serviços Principais

  • App Service: Hospedagem de aplicações web escaláveis (.NET, Java, Node.js, etc.)
  • AKS (Azure Kubernetes Service): Kubernetes gerenciado na nuvem
  • Azure Functions: Execução serverless para funções e eventos
  • Azure DevOps: Plataforma completa para CI/CD, repositórios e gerenciamento de projetos

Pipelines de Integração Contínua (CI)

Automatizam build, testes e deploy para garantir qualidade e agilidade no desenvolvimento.

  • GitHub Actions: Workflows flexíveis integrados ao GitHub para CI/CD

Caso Real: Pipeline de CI do Itaú com Azure DevOps

Boas Práticas em Integração Contínua

  • Automatizar o build e testes para cada commit
  • Incluir análise estática de código (ex: SonarQube)
  • Manter pipelines rápidas para feedback imediato
  • Versionar configurações de pipeline junto ao código
  • Integrar com ambientes de homologação para testes integrados

Exemplo de Pipeline YAML no Azure DevOps

trigger:
  branches:
    include:
      - main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: Maven@3
  inputs:
    mavenPomFile: 'pom.xml'
    goals: 'clean package'
- task: PublishBuildArtifacts@1
  inputs:
    PathtoPublish: 'target'
    ArtifactName: 'drop'

Exemplo de GitHub Actions para Node.js

name: Node.js CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Use Node.js 16.x
      uses: actions/setup-node@v3
      with:
        node-version: '16'
    - run: npm install
    - run: npm test

SonarQube para Análise Estática e Qualidade de Código

SonarQube é uma ferramenta que analisa código fonte para detectar bugs, vulnerabilidades e code smells.

  • Suporte a múltiplas linguagens: Java, C#, JavaScript, entre outras
  • Integração com pipelines de CI para bloqueio automático em falhas
  • Relatórios detalhados com métricas e tendências de qualidade

Integrando SonarQube no Azure DevOps Pipeline

steps:
- task: SonarQubePrepare@5
  inputs:
    SonarQube: 'SonarQubeServiceConnection'
    scannerMode: 'MSBuild'
    projectKey: 'myProjectKey'
    projectName: 'My Project'
- task: VSBuild@1
  inputs:
    solution: '**/*.sln'
- task: SonarQubeAnalyze@5
- task: SonarQubePublish@5
  inputs:
    pollingTimeoutSec: '300'

Exercício: Criar Pipeline CI com GitHub Actions

Objetivo:

Configurar um pipeline CI no GitHub Actions para seu projeto que execute:

  2. Execução dos testes automatizados
  3. Análise estática de código usando SonarCloud ou SonarQube

Exemplo: Pipeline CI para Java Spring Boot

name: Java CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Set up JDK 17
      uses: actions/setup-java@v3
      with:
        java-version: '17'
    - name: Build with Maven
      run: mvn clean package --file pom.xml
    - name: Run tests
      run: mvn test
    - name: SonarQube Scan
      uses: sonarsource/sonarcloud-github-action@v1
      with:
        projectBaseDir: '.'

Exemplo: Pipeline CI para .NET C#

trigger:
- main

pool:
  vmImage: 'windows-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '6.0.x'
- script: dotnet build --configuration Release
  displayName: 'Build project'
- script: dotnet test --no-build --verbosity normal
  displayName: 'Run tests'
- task: SonarQubePrepare@5
  inputs:
    SonarQube: 'SonarQubeServiceConnection'
    scannerMode: 'MSBuild'
    projectKey: 'dotnet-project'
    projectName: 'DotNet Project'
- task: SonarQubeAnalyze@5
- task: SonarQubePublish@5
  inputs:
    pollingTimeoutSec: '300'
