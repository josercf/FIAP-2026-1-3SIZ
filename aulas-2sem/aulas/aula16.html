Continuous Deployment (CD)

Deploy Contínuo é a prática de automatizar o lançamento de software para ambientes de produção com segurança e rapidez.

Permite entregas frequentes, reduzindo riscos e aumentando a eficiência do time.

16 - Deploy Continuo + Infrastructure as Code
5

Estratégias de Deploy

  • Blue-Green: Dois ambientes idênticos, um ativo e outro preparado para deploy.
  • Canary: Deploy gradual para pequenos grupos de usuários para validação.
  • Rolling: Atualização gradual do ambiente, substituindo instâncias aos poucos.
  • Recreate: Desliga o sistema antigo e sobe o novo, com downtime.

6

Blue-Green Deployment

Dois ambientes idênticos: Blue (ativo) e Green (novo deploy).

Após testes, o tráfego é redirecionado para o Green, tornando-o ativo.

Garante rollback rápido e zero downtime.

7

Canary Deployment

Deploy gradual para uma pequena parte dos usuários (canário).

Monitoramento detalhado para identificar falhas antes do rollout completo.

Reduz riscos de falhas em produção.

16 - Deploy Continuo + Infrastructure as Code
8

Rolling e Recreate Deployment

  • Rolling: Atualiza instâncias gradualmente sem downtime total.
  • Recreate: Para o sistema antigo e inicia o novo, causando downtime.

Rolling é preferido para sistemas críticos com alta disponibilidade.

Infrastructure as Code (IaC)

Prática de gerenciar infraestrutura via código declarativo ou imperativo.

Permite versionamento, automação e replicação de ambientes com segurança.

Principais ferramentas:

  • Terraform (HCL)
  • Ansible (YAML)
  • Bicep (Azure)

Terraform - HCL (HashiCorp Configuration Language)

Ferramenta declarativa para provisionar infraestrutura em múltiplos provedores.

provider "azurerm" {
  features {}
}

resource "azurerm_resource_group" "rg" {
  name     = "fiap-rg"
  location = "East US"
}

GitOps

Prática onde o Git é a fonte única da verdade para infraestrutura e aplicações.

Automatiza deploys e sincroniza estado do ambiente com o repositório Git.

Ferramentas populares:

  • ArgoCD
  • Flux

ArgoCD e Flux

ArgoCD: Ferramenta declarativa para Kubernetes, sincroniza estado do Git com cluster.

Flux: Automatiza entrega contínua Kubernetes, monitora repositórios Git.

Exercício Hands-On

Crie um template Terraform básico para provisionar um App Service no Azure

Passos para o exercício:

  1. Defina o provider Azure no arquivo main.tf.
  2. Crie um resource group chamado fiap-rg.
  3. Configure um App Service Plan básico.
  4. Declare o recurso do App Service vinculado ao App Service Plan e Resource Group.
  5. Valide e aplique o template localmente ou via Azure CLI.

Exemplo inicial para começar:

provider "azurerm" {
  features {}
}

resource "azurerm_resource_group" "fiap" {
  name     = "fiap-rg"
  location = "eastus"
}

Continue definindo o azurerm_app_service_plan e azurerm_app_service.
