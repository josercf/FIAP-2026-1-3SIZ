Conceitos Chave: API Gateway & Load Balancer

API Gateway: Ponto único de entrada para microserviços, responsável por roteamento, autenticação, e agregação de chamadas.

Load Balancer: Distribui requisições entre múltiplas instâncias para garantir alta disponibilidade e escalabilidade.


// Exemplo simples de API Gateway com Spring Cloud Gateway
@Bean
public RouteLocator customRouteLocator(RouteLocatorBuilder builder) {
  return builder.routes()
    .route("lb_route", r -> r.path("/api/**")
      .uri("lb://MICROSERVICE"))
    .build();
}

Testes, Cache & Filas

  • Testes: Unitários, integração, contrato e end-to-end para garantir qualidade.
  • Cache: Acelera respostas usando armazenamento temporário, ex: Redis.
  • Filas: Comunicação assíncrona entre serviços para desacoplamento e escalabilidade.

// Exemplo Node.js com Redis cache
const redis = require('redis');
const client = redis.createClient();

async function getUser(id) {
  const cache = await client.get(`user:${id}`);
  if(cache) return JSON.parse(cache);
  const user = await fetchUserFromDb(id);
  await client.set(`user:${id}`, JSON.stringify(user), {EX: 3600});
  return user;
}

Service Mesh com Istio e Linkerd

Service Mesh é uma camada dedicada à comunicação entre microserviços, oferecendo observabilidade, segurança e gerenciamento de tráfego.

Istio e Linkerd são as ferramentas líderes, facilitando políticas, telemetria e resiliência.


# Exemplo básico configuração Istio VirtualService
apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: reviews
spec:
  hosts:
  - reviews.prod.svc.cluster.local
  http:
  - route:
    - destination:
        host: reviews
        subset: v1

Exercício Hands-on

Implementando Resiliência com Circuit Breaker em Spring Boot

Objetivo: implementar o padrão Circuit Breaker usando Resilience4j no serviço Spring Boot.

  1. Adicione a dependência do Resilience4j no pom.xml.
  2. Configure o Circuit Breaker na classe de configuração.
  3. Anote o método que realiza chamadas externas com @CircuitBreaker.
  4. Teste o comportamento simulando falhas.


<dependency>
  <groupId>io.github.resilience4j</groupId>
  <artifactId>resilience4j-spring-boot2</artifactId>
  <version>1.7.1</version>
</dependency>
          

@Configuration
public class ResilienceConfig {
  @Bean
  public Customizer<CircuitBreakerConfigBuilder> defaultCustomizer() {
    return builder -> builder
      .failureRateThreshold(50)
      .waitDurationInOpenState(Duration.ofSeconds(10));
  }
}
          

@Service
public class ExternalService {
  @CircuitBreaker(name = "backendService", fallbackMethod = "fallback")
  public String callExternalApi() {
    // lógica da chamada externa
  }

  public String fallback(Throwable t) {
    return "Fallback response";
  }
}
