Pirâmide de Testes

Estratégia para balancear testes em diferentes níveis:

  • Testes de Unidade: pequenos, rápidos, isolados.
  • Testes de Integração: verificam interação entre componentes.
  • Testes End-to-End (E2E): validam o sistema completo do ponto de vista do usuário.

TDD (Test Driven Development)

Fluxo em três etapas para desenvolvimento orientado a testes:

  1. Red: escrever um teste que falha.
  2. Green: implementar código mínimo para passar no teste.
  3. Refactor: melhorar o código mantendo os testes verdes.

Mocks, Stubs e Fakes

Objetos utilizados para simular comportamentos e dependências em testes:

  • Stub: retorna dados pré-definidos para chamadas específicas.
  • Mock: objeto que verifica se métodos foram chamados corretamente.
  • Fake: implementação simplificada que funciona para testes.

Dependency Injection (DI) para Testabilidade

DI facilita testes ao permitir que dependências sejam substituídas por mocks/stubs:

  • Promove código desacoplado e modular.
  • Permite injetar implementações fake na classe testada.
  • Frameworks comuns: Spring Boot, .NET Core DI, inversify.js (Node.js).

Diferença entre Testes de Unidade e Integração

  • Teste de Unidade: Verifica o funcionamento isolado de uma unidade (função, método).
  • Teste de Integração: Valida a interação entre múltiplos componentes ou sistemas.
  • Testes de integração geralmente envolvem banco de dados, APIs, serviços externos ou microsserviços.
  • Mais complexos e mais lentos que testes de unidade, porém essenciais para garantir a integração correta.

Testcontainers

Rodando banco de dados em container para testes

Testcontainers é uma biblioteca que permite rodar containers Docker para bancos e serviços durante testes automatizados.

  • Garante ambiente isolado e consistente para testes de integração.
  • Evita problemas de configuração de banco local ou falta de dados.
  • Suporta diversos bancos: PostgreSQL, MySQL, MongoDB, Kafka, etc.

Contract Testing com Pact

Contract Testing é uma técnica para validar contratos entre serviços (consumidores e provedores).

  • Pact é uma ferramenta popular para testes de contrato.
  • Garante que serviços independentes evoluam sem quebrar integrações.
  • Evita testes end-to-end caros e frágeis.

// Exemplo de contrato Pact (simplificado)
{
  "consumer": { "name": "frontend" },
  "provider": { "name": "user-service" },
  "interactions": [
    {
      "description": "Busca usuário por ID",
      "request": {
        "method": "GET",
        "path": "/users/123"
      },
      "response": {
        "status": 200,
        "body": {
          "id": 123,
          "name": "João"
        }
      }
    }
  ]
}

Exemplo Java/Spring Boot

Testcontainers + RestAssured


import org.junit.jupiter.api.Test;
import org.testcontainers.containers.PostgreSQLContainer;
import io.restassured.RestAssured;
import static io.restassured.RestAssured.*;
import static org.hamcrest.Matchers.*;

public class UserIntegrationTest {

  static PostgreSQLContainer postgreSQLContainer = new PostgreSQLContainer<>("postgres:14")
    .withDatabaseName("testdb")
    .withUsername("test")
    .withPassword("test");

  static {
    postgreSQLContainer.start();
    System.setProperty("DB_URL", postgreSQLContainer.getJdbcUrl());
    System.setProperty("DB_USER", postgreSQLContainer.getUsername());
    System.setProperty("DB_PASS", postgreSQLContainer.getPassword());
  }

  @Test
  public void testCreateAndGetUser() {
    RestAssured.baseURI = "http://localhost:8080";

    // Cria usuário
    given()
      .contentType("application/json")
      .body("{\"name\":\"Maria\"}")
    .when()
      .post("/users")
    .then()
      .statusCode(201)
      .body("name", equalTo("Maria"));

    // Consulta usuário
    when()
      .get("/users")
    .then()
      .statusCode(200)
      .body("name", hasItem("Maria"));
  }
}

Exemplo .NET/C#

Testcontainers.DotNet + xUnit + HttpClient


using System.Net.Http;
using System.Text;
using System.Threading.Tasks;
using Xunit;
using DotNet.Testcontainers.Builders;
using DotNet.Testcontainers.Containers;

public class UserIntegrationTests : IAsyncLifetime {
  private readonly TestcontainerDatabase _postgresContainer;
  private readonly HttpClient _client = new HttpClient { BaseAddress = new Uri("http://localhost:5000") };

  public UserIntegrationTests() {
    _postgresContainer = new TestcontainersBuilder()
      .WithDatabase(new PostgreSqlTestcontainerConfiguration {
        Database = "testdb",
        Username = "test",
        Password = "test"
      }).Build();
  }

  public async Task InitializeAsync() {
    await _postgresContainer.StartAsync();
    // Configure a string de conexão na aplicação
  }

  public async Task DisposeAsync() {
    await _postgresContainer.StopAsync();
  }

  [Fact]
  public async Task TestCreateAndGetUser() {
    var json = "{\"name\":\"Carlos\"}";
    var content = new StringContent(json, Encoding.UTF8, "application/json");

    var postResponse = await _client.PostAsync("/users", content);
    Assert.True(postResponse.IsSuccessStatusCode);

    var getResponse = await _client.GetAsync("/users");
    var responseBody = await getResponse.Content.ReadAsStringAsync();
    Assert.Contains("Carlos", responseBody);
  }
}

Exemplo Node.js

testcontainers + supertest + Jest


const { PostgreSqlContainer } = require('testcontainers');
const request = require('supertest');
const app = require('../app');

let container;

beforeAll(async () => {
  container = await new PostgreSqlContainer('postgres:14')
    .withDatabase('testdb')
    .withUsername('test')
    .withPassword('test')
    .start();

  process.env.DB_HOST = container.getHost();
  process.env.DB_PORT = container.getPort();
  process.env.DB_USER = 'test';
  process.env.DB_PASS = 'test';
  process.env.DB_NAME = 'testdb';

  await app.connectDatabase(); // exemplo de método para conectar o DB
});

afterAll(async () => {
  await container.stop();
});

test('POST /users cria usuário e GET /users retorna lista', async () => {
  await request(app)
    .post('/users')
    .send({ name: 'Ana' })
    .expect(201);

  const res = await request(app).get('/users').expect(200);
  expect(res.body).toEqual(
    expect.arrayContaining([
      expect.objectContaining({ name: 'Ana' }),
    ])
  );
});
